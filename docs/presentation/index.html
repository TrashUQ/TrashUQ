TrashUQ

Edge AI + Federated Learning for Real-Time Trash Classification Monitoring

Edge-to-Cloud Monitoring Platform with Deployed Validation

Formal Project Presentation

Advanced Engineering & Federated Systems

Motivation

Urban Waste Challenges

Traditional waste management lacks real-time visibility into classification accuracy and node health at the edge.

Edge Intelligence

Processing data at the source (Arduino-class nodes) reduces latency and bandwidth usage compared to cloud-only processing.

Collaborative Learning

Federated Learning (FL) allows nodes to improve global models without sharing raw, sensitive data.

Operational Visibility

A centralized dashboard provides real-time telemetry and monitoring of the entire edge-to-cloud pipeline.

Project Goals

  • Edge Classification: Real-time material identification on constrained hardware.
  • Live Telemetry: Seamless ingestion of metrics and events via MQTT.
  • Persistent Backend: Robust storage for historical analysis and traceability.
  • Real-Time Dashboard: Centralized observability of fleet status and FL progress.
  • Federated Coordination: Managed global model updates with concurrency handling.
  • Multi-Client Validation: Rigorous testing on physical Arduino-class hardware.

Edge Layer

The system utilizes Arduino-class devices as primary edge nodes.

  • Hardware: Arduino UNO Q MPU series.
  • Telemetry: MQTT-based event and metric streaming.
  • Inference: TFLite-based material classification path.
  • Validation: Controlled synthetic frames for isolating pipeline paths.

Federated Learning Coordination

gRPC Protocol

  • Join: Registration of new edge nodes.
  • GetGlobalModel: Fetching current model head.
  • SubmitUpdate: Pushing local gradients/weights.

Algorithm

  • FedAvg: Federated Averaging.
  • Monotonic model versioning (0 → N).
  • Synchronous aggregation rounds.

Concurrency & Version Control

In a distributed system, parallel submissions can lead to race conditions. TrashUQ implements Monotonic Versioning to protect model consistency.

The Challenge

Nodes may train on v25 while the coordinator has already progressed to v26.

The Solution

Stale updates are detected and correctly rejected. During validation, 5 stale parallel updates were successfully handled.

Lessons Learned

  • MQTT Scalability: Highly effective for constrained edge node telemetry.
  • Persistence: Centralized SQL state is essential for auditability.
  • Observability: Real-time dashboards significantly improve debugging efficiency.
  • Version Control: Monotonic versioning is a prerequisite for robust FL coordination.
  • Controlled Validation: Synthetic frame testing ensures pipeline stability before field deployment.

Conclusion

TrashUQ demonstrates a validated Edge AI + Federated Learning platform capable of real-time monitoring and collaborative model improvement.

  • End-to-end integration from Arduino-class hardware to cloud dashboards.
  • Verified robustness against message loss and concurrency races.
  • Scalability evidence supporting multi-node federated deployments.
